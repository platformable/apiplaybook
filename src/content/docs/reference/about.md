title: About our Context-first governance approach
description: An overview of context first governance
---

Context-first governance is about understanding and describing intent in AI native digital systems.

It brings in an understanding of: 
- Use cases
- The wider operational environment
- The organizational structure and 
- Business capabilities.

It address common challenges in composable architecture including: 
- API drift: where the API does not perform the functions it was created to delver
- API sprawl: where multiple APIs are built for similar functionality, often with duplicative data models, and
- API complexity: where APis are each built in different ways and described in different ways with different data models, requiring high cognitive load to use multiple APIs from your organisation.

Context-first governance addresses data needs and usage, making sure it's done within compliance requirements. 

And at the end of the day, context-first governance is trying to support **value-based** product and service delivery.
